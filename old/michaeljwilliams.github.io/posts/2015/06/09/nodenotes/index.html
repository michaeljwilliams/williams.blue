Notes On The Notes

A little while ago I decided to read through the whole Node.js API and see what useful bits I could find. In order to have an easily-digested overview of the API I wrote these notes on all the modules and some methods I thought would come in handy.

The structure of the notes follows the Node.js API docs pretty closely. I’ve left out parts which I thought weren’t very useful (just my opinion, based on what I do), and added some short explanations for what many of them do or can be used for. I also included the global objects which aren’t modules, such as __filename.

The name of each module is followed by its scope in parantheses. Objects that are ‘global’ are exactly that—they and their methods are always available. Those that are ‘module global’ are available without being required but are specific to each module (think module as in npm module—the project you’re working on; ex. __filename (two underscores) is specific to the code being executed). ‘Require’ means you need to require(); that module if you want to use it.

Most modules just have a short explanation for their usage. Others have a list of use cases and others still have a list of specific methods I thought would be useful and wanted to remember (method names are preceded by a dot). Without further ado:

assert (require)

  • for tests

Buffer (global)

  • for non-text (binary) data

C/C++ addons (require ./)

  • glue to C and C++ libraries (eg V8 library)
  • files or directories of your C or C++ libraries with respect to current directory ex. require('./build/Release/addon');

child_process (require)

  • create child processes (eg run a process in a shell)

cluster (require)

  • node is single threaded; use this to run a cluster of node processes

console (global)

  • console.log: printing to stdout and stderr

crypto (require)

  • hashes
  • OpenSSL methods
  • HTTPS methods

debugger (global?)

  • V8 debugger accessible via TCP

dns (require)

  • DNS lookup via operating system
  • DNS lookup via DNS server

domain (require)

  • use with cluster
  • handle multiple different I/O operations as a single group
  • Domain object will be notified of any errors; context of error is kept

events (require)

  • for emitting events
  • objects which emit events are instances of events.EventEmitter
  • event listeners

__filename (module global)

  • filename of code being executed
  • note that there are two underscores in the name of this object

__dirname (module global)

  • name of directory that __filename resides in
  • note that there are two underscores in the name of this object

fs (require)

  • file I/O; all methods have sync and async forms
  • .rename, .open, .close, .read, .write, .readFile, .writeFile, .mkdir, .rm, .rmdir, .chown, .chmod, .truncate, .append, .watch, .exists

http, https (require)

  • create server
  • create client
  • http request, http get

module (module global)

  • reference to object representing current module
  • module.exports: expose functions etc outside module scope
  • module.require: use require(); as if called from original module
  • .id, .filename, .loaded, .parent, .children

net (require)

  • async network wrapper (streams); both servers and clients
  • create server, create connection, create socket, etc

os (require)

  • basic os info utilities
  • .tmpdir, .endianness, .hostname, .type, .platform, .arch, .release, .uptime, .loadavg, .totalmem, .freemem, .cpus, .networkInterfaces, .EOL

path (require)

  • parsing and manipulating file paths
  • .normalize, .join, .resolve, .isAbsolute, .relative, .dirname, .basename, .extname, .sep, .delimiter, .parse, .format

process (global)

  • instance of EventEmitter
  • use to output to and accept input from CLI
  • .stdin, .stdout, .stderr, .exec, .abort, .exit, .exitCode, .cwd, .env, .kill, .pid, .title, .uptime, .mainModule

punycode (require)

  • ascii to/from unicode encoding/decoding
  • ucs2 encoding/decoding

querystring (require)

  • utilities for dealing with query strings
  • .stringify, .parse, .escape, .unescape

readline (require)

  • read a stream such as process.stdin line by line

repl (require)

  • REPL provides a way to interactively run JavaScript and see the results
  • used for debugging, testing, or just trying things out

require (module global)

  • requires modules
  • .resolve exact location of file
  • .cache: delete key value from require.cache to reload module on next require
  • require('/foo'); = absolute path; require('./foo'); or require('../foo'); = relative; require('foo'); = core module or from node_modules folder

smalloc (global ?) experimental

  • simple allocator; raw memory allocation; used by Buffers

stream (require)

  • instance of EventEmitter
  • readable, writable, duplex (both), or transform (output computed from input)
  • .read, .resume, .pause, .isPaused, .close, .end, .pipe, .unpipe, .setEncoding

string_decoder (require)

  • decodes buffer to string (same as buffer.toString() but supports utf8)
  • .write, .end

Timers (all of following are global)

  • setTimeout(callback, delay[, arg][, ...]) run callback after delay milliseconds. End with clearTimeout(timeoutObject)
  • setInterval(callback, delay[, arg][, ...]) run callback every delay milliseconds. End with clearInterval(intervalObject)
  • setImmediate(callback[, arg][, ...]) immediate execution of callback after I/O events callbacks and before setTimeout/setInterval. End with clearImmediate(immediateObject)
  • FYI timers may not execute in exactly delay milliseconds - “Node.js makes no guarantees about the exact timing of when the callback will fire, nor of the ordering things will fire in. The callback will be called as close as possible to the time specified.”

tls (require)

  • uses OpenSSL to provide TLS and/or SSL: encrypted streams

tty (require)

  • terminal emulation
  • tty.ReadStream and tty.WriteStream classes

dgram (require)

  • User Datagram Protocol (UDP)
  • datagram socket creation etc

url (require)

  • .parse (href, protocol, slashes, host, auth, hostname, port, pathname, search, path, query, hash)
  • .format
  • .resolve

util (require)

  • supports Node’s internal APIs
  • .format: format strings
  • .log: timestamped message to stdout
  • .inspect: return a string representation of object, useful for debugging
  • .isArray, .isRegExp, .isDate, .isError, .inherits

vm (require)

  • JavaScript code can be compiled and run immediately or saved and run later
  • sandboxing

zlib (require)

  • Gzip/Gunzip, Deflate/Inflate, and DeflateRaw/InflateRaw classes (all readable/writable Streams)
  • Compressing or decompressing a file can be done by piping an fs.ReadStream into a zlib stream, then into an fs.WriteStream
